Project Background

About This Project

What This Is

This is an open-source computer vision pipeline for extracting and visualizing the three-dimensional facial geometry encoded in the Shroud of Turin's image. The project builds on the 1976 VP-8 Image Analyzer discovery (Jackson, Jumper, Mottern) and extends it using modern depth map processing, landmark detection, and depth-constrained AI generation.

All code is written in Python, all parameters are documented, and all outputs are framed as data rather than conclusions. The pipeline is designed to be reproducible by anyone with access to the source images and a GPU capable of running Stable Diffusion.

Who Built This

This project was built by a Catholic researcher who is personally convinced that the Shroud is the authentic burial cloth of Jesus Christ. That conviction motivated the project but does not direct its outputs. The pipeline is engineered to produce the same results regardless of what anyone believes about the Shroud's origin.

The goal is to let the data speak. If the data is wrong, I want to know. If the data is right, I want others to be able to verify it.

Scope Statement

This project does not:

  • Make claims about the Shroud's authenticity
  • Assert the identity of the man depicted
  • Claim the AI reconstructions are portraits of Jesus
  • Make theological arguments of any kind

This project does:

  • Extract and measure the 3D geometry the Shroud encodes
  • Apply the same methods forensic anthropologists use with burial remains
  • Cross-validate findings across two independent source photographs
  • Document all decisions, uncertainties, and limitations honestly

A Note on the Images

The AI reconstructions on this site are depth-constrained renders of the facial geometry extracted from the Shroud. They are constrained by data. They are not portraits, and the pipeline makes no claim that they resemble any historical individual.

The sculptural material choice (clay, sandstone, marble) is deliberate: it removes pigmentation from the output entirely. The Shroud does not encode color. Any skin tone applied to a reconstruction would be an assumption, not a finding.

Open Source

All source code, pipeline scripts, and parameter documentation are available for inspection and replication. The project is designed so that any researcher with access to the same source images and hardware can reproduce every output shown on this site.

Source images (Enrie 1931, Miller 1978 STURP) are not redistributed here but are publicly available from their respective archives.

Honest Limitations

  • The 150×150 downsample loses fine facial detail. Nose length measurement (bridge to tip) is unreliable at this resolution.
  • Scale calibration depends on IPD reference (6.3 cm adult male average). Individual variation could shift all measurements proportionally by ±10–15%.
  • Cloth draping at the time of burial affects the depth encoding — particularly the vertical axis. Some vertical measurements (nose-to-chin) may be elongated by 10–15% relative to true anatomy.
  • The healed (symmetrized) depth map and its reconstructions are analytical tools, not predictions. The specific nature of any injuries or trauma cannot be determined from this analysis.
  • MediaPipe FaceLandmarker was tested and failed completely. Our custom algorithm produces good results but has not been formally validated against a ground-truth face dataset with known measurements.
